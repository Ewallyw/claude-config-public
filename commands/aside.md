# /aside

临时中断当前任务，处理用户的插入问题，并在回答后可继续原任务。

## 用法

```text
/aside <问题或请求>
```

## 执行规则

1. 先用 1-3 条要点记录当前任务上下文（目标、进度、下一步）。
2. 聚焦回答插入问题，不扩展到无关实现。
3. 回答结束后明确给出恢复提示：
   - `输入“继续”或“/resume”恢复原任务`。
4. 若插入请求要求永久改计划，先确认，再更新原任务计划。

## 约束

- 不丢失原任务上下文。
- 除非用户明确要求，不在 aside 流程中提交代码或改动任务范围。
- 插入问题涉及高风险操作时，先给出安全边界和确认步骤。
