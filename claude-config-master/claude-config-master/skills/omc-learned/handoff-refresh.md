# 会话交接刷新

## 触发条件
一个重大 milestone 完成后（阶段代码完成、文档整理完成、文献调研完成、论文章节完成等），需要将当前状态固化到 `handoff.md` 供新会话读取。

## 前置：判断是否需要刷新
满足以下任一条件时执行：
- 项目目录结构变更（增/删/移动/重命名目录或关键文件）
- 核心结论更新（实验数据、叙事逻辑变化）
- 文档一致性状态变化（哪些文档已/未对齐权威源）
- 新增或完成待办事项
- AI 协作体系变更（MCP 服务器、Skill 增删）
- 距上次刷新超过 3 个重大变更

## 流程

### Step 1：核实当前项目状态
```bash
# 根目录
ls "项目根目录/"

# 关键子目录
ls "docs/"
ls "Overall/"
ls "论文资料/"  # 或其他资料目录

# 新产生的计划/结果文件
ls ".omc/plans/"
```
**原则**：不依赖记忆，用 ls/glob 确认实际存在哪些文件。

### Step 2：读取现有 handoff.md
了解当前交接文档写到了哪里，哪些需要更新、哪些保留。

### Step 3：汇总本会话变更
从以下维度整理：
1. **目录结构变更** — 增删移了什么，最终结构什么样
2. **文档修正** — 哪些文件改了、为什么改、对齐了什么权威源
3. **新增产出** — 计划文件、结果文件、Skill 文件、配置变更
4. **结论更新** — 核心实验结论有无变化（数值、叙事方向）
5. **文献/引用变更** — 新增/删除/更新了什么引用

### Step 4：确认权威源层级
handoff 中实验结论的权威源必须是：
```
results/ 下的实际数据文件（.md 表格 > .npz 数据 > .png 图表）
    ↑ 衍生自
验证结论-2026-05-27.md（成立/部分成立/不成立的判断）
    ↑ 映射到
论文引用库.md（代码位置 + 证据路径）
```
handoff 不创造新结论，只引用已有权威源。

### Step 5：写 handoff.md（覆盖模式）
用以下 10 节结构（根据实际情况调整节数）：

```
1. 你是谁/你在哪 — 项目身份+本会话完成事项
2. 项目结构（最终版）— 带注释的目录树
3. AI 协作体系 — MCP 服务器 + Skill 速查
4. 核心结论（权威参考）— 实验数据表+叙事定位
5. 文档一致性状态 — 每个文件群的同步状态
6. 文献与引用状态 — 已有/新增/缺失清单
7. 写作/工作关键入口 — 必读文件+援引链
8. 快速验证命令 — 一键运行的命令
9. 已知待办 — 优先级排序
10. 关键文件索引 — 路径→用途映射表
```

### Step 6：路径验证
handoff 中每条文件路径引用必须用 `[ -e "路径" ]` 或 `ls` 验证真实存在。
**零容忍断裂引用。**

## 反模式（不要做）
- ❌ 从记忆中写目录结构，不核实实际文件
- ❌ 引用不存在的路径（如旧文件名）
- ❌ handoff 中的结论与权威源矛盾
- ❌ 遗漏本会话的重大变更
- ❌ 写太长的背景叙事——handoff 是速查卡，不是日记
- ❌ 把 handoff 写成 AGENTS.md 的副本——两者分工：AGENTS.md 管规范，handoff 管状态

## 验收标准
- [ ] 新会话打开 handoff 后 5 分钟内能理解项目布局
- [ ] 所有路径引用已验证存在
- [ ] 核心结论与权威源一致
- [ ] 本会话的变更无遗漏
- [ ] 下一阶段行动明确
