# 安全指南（中文）

## 核心原则

1. **最小权限**：只执行完成任务所需的最小操作。
2. **最小改动**：仅修改与需求直接相关的文件与逻辑。
3. **默认不信任输入**：所有外部输入都必须验证和约束。
4. **不暴露敏感信息**：禁止输出、提交或记录密钥、令牌、凭据、隐私数据。

## 开发与修改要求

- 涉及命令执行、文件路径、网络请求时，优先做白名单校验。
- 禁止引入危险模式（如未转义拼接命令、宽松权限、硬编码凭据）。
- 变更后必须执行现有验证（测试/构建/检查）并记录结果。
- 若发现与本次改动耦合的安全风险，必须一并修复或明确阻塞原因。

## 审查清单

- 是否新增了可被注入的输入通道？
- 是否把敏感数据写入代码、日志或文档？
- 是否扩大了文件系统、网络或执行权限范围？
- 是否对失败路径做了安全处理（拒绝、回滚、告警）？

## 提交前门槛

- 无新增凭据与敏感信息。
- 无明显注入、越权、数据泄露风险。
- 验证步骤已执行，结论可复现。
